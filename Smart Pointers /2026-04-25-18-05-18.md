## Rust Interior Mutability

Rust's ownership system enforces a simple rule at compile time: you can have *either* one mutable reference *or* any number of immutable references — never both. **Interior mutability** is a design pattern that bends this rule safely by moving the borrow-checking from compile time to runtime (or using special unsafe primitives), all while maintaining memory safety guarantees through the type system.

---

### The Core Principle: `UnsafeCell<T>`

Every interior mutability type in Rust is ultimately built on `UnsafeCell<T>`, the only way in Rust to legally obtain a mutable pointer from a shared reference:

```rust
pub struct UnsafeCell<T: ?Sized> {
    value: T,
}

impl<T> UnsafeCell<T> {
    // The magic primitive: &self -> *mut T
    pub fn get(&self) -> *mut T {
        self as *const UnsafeCell<T> as *mut T
    }
}
```

`UnsafeCell` is a **lang item** — the compiler knows about it specifically. Without it, turning `&T` into `*mut T` would be undefined behavior. With it, the compiler opts out of certain alias-based optimizations, making mutation through a shared reference sound. Everything else (Cell, RefCell, Mutex) is built as a safe abstraction layer on top.

---

### `Cell<T>` — Copy-based, Zero-overhead

`Cell<T>` allows mutation through a shared reference but only for `Copy` types (or via whole-value replacement). It is **not** `Sync`, so it can never be shared across threads.

**Internal structure:**
```rust
pub struct Cell<T: ?Sized> {
    value: UnsafeCell<T>,
}
```

**How `get` and `set` work:**
```rust
impl<T: Copy> Cell<T> {
    pub fn get(&self) -> T {
        // SAFETY: safe because we copy the value out.
        // No reference to the interior is ever handed out.
        unsafe { *self.value.get() }
    }

    pub fn set(&self, val: T) {
        // SAFETY: safe because T: Copy — no destructor is run
        // on the old value through a live reference.
        unsafe { *self.value.get() = val; }
    }
}
```

**Why it's safe:**
The key invariant is that `Cell<T>` **never gives out a reference to its interior**. You can only copy values in or out. Because no `&T` or `&mut T` to the inner value ever exists simultaneously, aliasing rules are never violated. The single-threaded constraint (`!Sync`) ensures no data races.

**Typical use case:**
```rust
use std::cell::Cell;

struct Config {
    debug_count: Cell<u32>,  // mutate through & reference
}

impl Config {
    fn log(&self) {
        self.debug_count.set(self.debug_count.get() + 1);
    }
}
```

### `RefCell<T>` — Runtime Borrow Checking

`RefCell<T>` lifts Rust's borrow checker rules to runtime. It allows interior mutation for non-`Copy` types by tracking borrows with a counter at runtime, panicking if the rules are violated.

**Internal structure:**

```rust
pub struct RefCell<T: ?Sized> {
    borrow: Cell<BorrowFlag>,  // runtime borrow counter
    value: UnsafeCell<T>,
}

// Positive = number of active immutable borrows
// -1       = one active mutable borrow
type BorrowFlag = isize;
```

**How `borrow` and `borrow_mut` work:**
```rust
impl<T> RefCell<T> {
    pub fn borrow(&self) -> Ref<'_, T> {
        let b = self.borrow.get();
        assert!(b >= 0, "already mutably borrowed");  // simplified
        self.borrow.set(b + 1);
        // Returns a guard that decrements the counter on Drop
        Ref { value: unsafe { &*self.value.get() }, borrow: &self.borrow }
    }

    pub fn borrow_mut(&self) -> RefMut<'_, T> {
        let b = self.borrow.get();
        assert!(b == 0, "already borrowed");          // simplified
        self.borrow.set(-1);
        RefMut { value: unsafe { &mut *self.value.get() }, borrow: &self.borrow }
    }
}
```

**The RAII guards (`Ref` and `RefMut`):**
```rust
impl Drop for Ref<'_, T> {
    fn drop(&mut self) {
        // Decrement the immutable borrow count
        self.borrow.set(self.borrow.get() - 1);
    }
}

impl Drop for RefMut<'_, T> {
    fn drop(&mut self) {
        // Clear the mutable borrow flag
        self.borrow.set(0);
    }
}
```

The guards encode borrow lifetime into ownership. When a `Ref` or `RefMut` is dropped, the count is updated. Violating the rules (two `borrow_mut` calls, or `borrow_mut` while `borrow` is active) causes a **runtime panic** rather than a compile error.

**Typical use case — graph/tree with back-references:**
```rust
use std::cell::RefCell;
use std::rc::Rc;

type NodeRef = Rc<RefCell<Node>>;

struct Node {
    value: i32,
    children: Vec<NodeRef>,
}

let root: NodeRef = Rc::new(RefCell::new(Node { value: 0, children: vec![] }));
root.borrow_mut().value = 42;  // mutate through shared Rc
```

### `Mutex<T>` — Thread-safe Interior Mutability

`Mutex<T>` extends the same idea to multi-threaded contexts. Instead of a borrow counter, it uses an **OS-level or kernel-backed lock** (futex on Linux, SRWLOCK on Windows) to ensure only one thread can access the data at a time.

**Internal structure (simplified):**
```rust
pub struct Mutex<T: ?Sized> {
    inner: sys::Mutex,    // platform-specific lock primitive
    poison: PoisonFlag,   // tracks if a thread panicked while holding the lock
    data: UnsafeCell<T>,
}
```

**How `lock` works:**
```rust
impl<T> Mutex<T> {
    pub fn lock(&self) -> LockResult<MutexGuard<'_, T>> {
        // Blocks the current thread until the OS grants the lock
        unsafe { self.inner.lock() };

        if self.poison.get() {
            Err(PoisonError::new(MutexGuard { lock: self }))
        } else {
            Ok(MutexGuard { lock: self })
        }
    }
}
```

**The RAII guard:**
```rust
impl<T> Deref for MutexGuard<'_, T> {
    type Target = T;
    fn deref(&self) -> &T {
        unsafe { &*self.lock.data.get() }
    }
}

impl<T> DerefMut for MutexGuard<'_, T> {
    fn deref_mut(&mut self) -> &mut T {
        unsafe { &mut *self.lock.data.get() }
    }
}

impl<T> Drop for MutexGuard<'_, T> {
    fn drop(&mut self) {
        // Release the OS lock
        unsafe { self.lock.inner.unlock() };
    }
}
```

**Poisoning:** If a thread panics while holding a `MutexGuard`, the mutex becomes *poisoned*. Subsequent `lock()` calls return an `Err(PoisonError)`. This is a safety mechanism: the data inside may be in an inconsistent state, so Rust forces you to explicitly acknowledge that before using it.

**Typical use case:**
```rust
use std::sync::{Arc, Mutex};
use std::thread;

let counter = Arc::new(Mutex::new(0u32));

let handles: Vec<_> = (0..8).map(|_| {
    let c = Arc::clone(&counter);
    thread::spawn(move || {
        *c.lock().unwrap() += 1;  // guard auto-released at end of scope
    })
}).collect();

for h in handles { h.join().unwrap(); }
println!("{}", *counter.lock().unwrap()); // 8
```

### Comparison Table

| Feature | `Cell<T>` | `RefCell<T>` | `Mutex<T>` |
|---|---|---|---|
| **Borrow check** | None (copy only) | Runtime (panic) | OS lock (block) |
| **Thread safe** (`Sync`) | ❌ No | ❌ No | ✅ Yes |
| **Gives out references** | ❌ No | ✅ Yes (guarded) | ✅ Yes (guarded) |
| **`T` constraint** | `Copy` (for `get`) | None | None |
| **Overhead** | Zero | 1 `isize` counter | OS syscall |
| **Failure mode** | N/A | Panic | Block / `PoisonError` |
| **Built on** | `UnsafeCell` | `UnsafeCell` + `Cell` | `UnsafeCell` + OS prim |

### Mental Model

Think of interior mutability as a **spectrum of trust**:

- **`Cell`** — "I trust myself to only copy, never alias." Zero cost, compiler enforces no aliasing structurally.
- **`RefCell`** — "I trust my logic but want the borrow checker to catch mistakes at runtime." Small counter overhead, panics on misuse.
- **`Mutex`** — "I share data across threads and need the OS to arbitrate access." Heavier, but the only sound choice across thread boundaries.

All three are sound because `UnsafeCell` gives the compiler a precise signal to suppress aliasing assumptions, and the abstractions above it ensure that at any moment, Rust's core invariant — **at most one live mutable reference to any data** — is upheld, whether at compile time, runtime, or through kernel serialization.
