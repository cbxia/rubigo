#" `&dyn Trait` vs `Box<dyn Trait>`

Both are **trait objects** — fat pointers containing a data pointer + vtable pointer — but they differ in ownership and lifetime.

---

### Core Difference

| | `&dyn Trait` | `Box<dyn Trait>` |
|---|---|---|
| Ownership | Borrowed (non-owning) | Owned |
| Allocation | No heap allocation | Heap-allocates the value |
| Lifetime | Tied to the borrow (`'a`) | Lives until dropped |
| Nullability | Never null | Never null |
| Size | 2 words (ptr + vtable) | 2 words (ptr + vtable) |

---

### `&dyn Trait` — Borrowed trait object

```rust
fn print_area(shape: &dyn Shape) {
    println!("{}", shape.area());
}

let circle = Circle { radius: 5.0 };
print_area(&circle); // circle still owned by caller
```

- The caller **retains ownership** of the concrete value
- No heap allocation required
- Requires an explicit lifetime (often elided): `&'a dyn Trait`
- Best for **read-only access** or short-lived polymorphism (function args)

---

### `Box<dyn Trait>` — Owned trait object

```rust
fn make_shape(kind: &str) -> Box<dyn Shape> {
    match kind {
        "circle" => Box::new(Circle { radius: 5.0 }),
        _        => Box::new(Square { side: 3.0 }),
    }
}

let shape = make_shape("circle"); // owns the value
```

- The `Box` **owns** the value and frees it on drop
- Necessary when you need to **return** a trait object, store it in a struct, or the concrete type isn't known at the call site
- Heap allocation is required
- Can be stored in collections: `Vec<Box<dyn Trait>>`

---

### When to use which

**Use `&dyn Trait` when:**
- You only need temporary, read-only (or borrowed) access
- The concrete value already exists somewhere and you don't want to move it
- Avoiding heap allocation matters (e.g. hot loops)

```rust
fn log(writer: &dyn Write) { ... }  // just borrows, no ownership needed
```

**Use `Box<dyn Trait>` when:**
- You need to **own** the trait object (return it, store it, move it)
- Building heterogeneous collections (`Vec<Box<dyn Trait>>`)
- The value must outlive the scope where it was created

```rust
struct App {
    handlers: Vec<Box<dyn Handler>>,  // owned, mixed concrete types
}
```

---

### Other variants worth knowing

- **`Rc<dyn Trait>`** — shared ownership, single-threaded
- **`Arc<dyn Trait>`** — shared ownership, thread-safe
- **`&mut dyn Trait`** — borrowed, mutable access without ownership

The rule of thumb: **prefer `&dyn Trait`** for function parameters where you don't need ownership; **use `Box<dyn Trait>`** when you need to own, store, or return the object.
